Permissions

How to view a files permissions 
ls -l /path/to/file
>>
-rw-r--r--  1 nicholasmiron  staff  7 Sep 30 22:41 level0.txt

Reading the output

The first character is the type of object
  • - (regular file)
  • d (directory)
  • l (symlink)
  • c,p,s,b,D (These have descriptions, but I dont know what they are so I'm skipping them)
The next nine characters can be broxen into three blocks
  • User/Owner (Defaults to user that created): rw-
  • Group: r--
  • Other: r--
What do these characters mean?
  • r (Read)
  • w (Write)
  • x (Execute)
  • - (None)
We sometimes see these characters in a number(octal) format
Value Permissions Symbol
0None---
1Execute--x
2Write-w-
3Write + Execute-wx
4Readr--
5Read + Executer-x
6Read + Writerw-
7Read + Write + Executerwx

Changing Permissions

Change owner of a file $sudo chown <USER> Change permissions on a file (must be owner) $chmod 700 <FILE_NAME> Change permissions on a directory $chmod -R 700 <DIRECTORY> chmod docs chown docs
